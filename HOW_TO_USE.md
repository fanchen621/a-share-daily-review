# 如何用好 A 股全时段智能复盘系统

> 写给 OpenClaw 的深度使用指南。不仅告诉你「怎么用」，更告诉你「怎么用好」。

---

## 一、理解这套系统的底层逻辑

### 1.1 它不是一个「数据罗列器」

市面上的复盘工具多如牛毛，但大多数只是把数据搬过来排个版。这套系统的核心设计理念是：

**数据采集 → 模式识别 → 经验积累 → 智能进化**

每一层都有明确的职责：

- **数据采集层**（a_share_scraper.py）：解决「数据从哪来」的问题。多源降级保证每天都能拿到数据，这是地基。
- **模式识别层**（SKILL.md 的分析框架）：解决「怎么解读数据」的问题。量能、趋势、情绪、板块轮动、量化策略——每个维度都是一个观察市场的窗口。
- **经验积累层**（data/lessons.md）：解决「上次学到的教训今天用上了吗」的问题。这是系统的「记忆」，没有它，每天的复盘都是从零开始。
- **智能进化层**（盘感自适应机制）：解决「分析方法要不要调整」的问题。市场风格会变，有效的信号也会变，系统需要自我调整。

### 1.2 三个时段的分工逻辑

为什么要分三个时段？因为每个时段要解决的问题完全不同：

**08:00 盘前速递** — 解决「今天要不要动」的问题
- 隔夜外盘告诉你全球风险偏好的方向
- 大宗商品告诉你通胀预期和避险情绪
- 新闻告诉你政策和事件催化剂
- 最终回答：今天整体方向是什么？该积极还是防守？

**12:00 午盘策略** — 解决「动什么、怎么动」的问题
- 上午的实际走势验证（或否定）了盘前判断
- 板块的真实资金流向比盘前猜测更可靠
- 连板梯队的实际情况告诉你市场情绪的真实温度
- 最终回答：哪些标的值得参与？什么价位介入？

**17:30 收盘复盘** — 解决「今天做对了什么、做错了什么」的问题
- 全天数据定格，可以做完整的归因分析
- 龙虎榜揭示了大资金的真实意图
- 量化营业部的行为模式为明天提供线索
- 最终回答：经验教训是什么？明天关注什么？

## 二、关键模块的深度使用

### 2.1 lessons.md — 这是系统的灵魂

很多人会忽略这个文件，觉得它只是个「记事本」。但它实际上是整个系统最重要的资产。

**为什么？**

因为 AI 模型本身没有「记忆」。每次对话都是独立的。如果不把经验写下来，每次复盘都等于从零开始——就像一个人每天失忆，每天重新学开车。

lessons.md 就是这个系统的「长期记忆」。它让系统能：
- 记住上次判断错了什么（判断修正）
- 记住哪些信号可靠（有效信号总结）
- 记住板块间的联动关系（板块联动规律）
- 记住量化资金的行为模式（量化营业部策略）

**怎么写好 lessons.md？**

1. **判断修正要诚实**。不要事后修改预判让它看起来「本来就是对的」。记录真实的偏差和原因，这才是有价值的。
2. **有效信号要量化**。不要写「连板梯队完整时市场通常比较好」，要写「连板梯队完整(首板→2板→3板) + 涨停>50家 → 近10次中8次次日上涨，中位涨幅1.2%」。
3. **板块联动要标注时滞**。不只记录「A涨了B也涨」，要记录「A涨了之后B在+1天/+2天跟涨」。
4. **量化策略要分类**。区分动量型/轮动型/回归型/事件驱动型，因为它们的应对策略完全不同。

### 2.2 量能分析 — 最被低估的维度

很多人看复盘只看涨跌幅，不看量能。但量能是验证趋势真实性的核心指标。

**关键用法：**

- **放量上涨** = 趋势有资金支撑，可信度高
- **缩量上涨** = 可能是存量博弈，后续乏力
- **放量下跌** = 恐慌出逃，但也是换手，可能很快见底
- **缩量下跌** = 阴跌，没有人接盘，最危险
- **量价背离** = 价格创新高但量能不跟，反转信号

系统会自动计算这些指标，但你需要理解它们的含义，才能做出正确判断。

### 2.3 量化营业部策略学习 — 核心特色

这是这套系统最独特的功能。大多数散户不关注龙虎榜中的量化席位，但它们其实蕴含着重要的信息。

**量化资金的特点：**
- 纪律性强，不会情绪化操作
- 策略相对固定，有模式可循
- 反应速度快，往往是第一个发现机会的

**怎么用好这个模块？**

1. **追踪量化席位的板块偏好**。如果量化连续3天买入新能源，说明他们的模型认为新能源有短期机会。
2. **区分量化的策略类型**。
   - 动量型：买入当日涨停/大涨的，次日通常还有溢价
   - 轮动型：从A撤出到B，预示板块风格切换
   - 回归型：买入超跌的，做反弹
3. **统计量化成功率**。如果近期量化买入的标的次日表现都很好，说明当前市场环境适合量化策略，散户跟着做也有汤喝。
4. **注意量化的卖出信号**。量化的卖出往往比买入更有信号价值——他们的风控模型比散户敏感。

### 2.4 情绪周期 — 什么时候该激进，什么时候该防守

市场情绪有周期性。这套系统用量化指标来定位当前处于哪个阶段：

| 阶段 | 特征 | 策略 |
|------|------|------|
| 冰点 | 涨停<20，跌停>30，最高板≤2 | 等待，不操作 |
| 修复 | 涨停20-40，出现3板 | 小仓位试错 |
| 发酵 | 涨停40-60，出现4-5板 | 积极参与主线 |
| 高潮 | 涨停>60，全线上涨 | 注意风险，准备减仓 |
| 退潮 | 涨停从高位回落>30% | 防守为主 |

**核心原则：冰点不割肉，高潮不追涨。**

系统的 lessons.md 会持续记录每次情绪周期的变化，逐渐形成更精准的周期判断能力。

### 2.5 盘感自适应 — 让系统「长大」

盘感不是玄学，它是大量经验的模式匹配。

这套系统的盘感机制是这样工作的：

1. **每次复盘后**，系统自动更新 lessons.md，记录新的经验
2. **下次分析时**，系统先读 lessons.md，把历史经验注入当前分析
3. **权重调整**：如果近期政策面对市场影响更大，系统会提升政策面维度的权重；如果近期技术面更有效，就提升技术面权重
4. **信号验证**：系统会验证历史上有效的信号在当前市场是否仍然有效

**重要：不要跳过更新 lessons.md 的步骤。** 每次复盘必须更新，否则系统不会成长。这就像一个人不写日记就不会反思一样。

## 三、三时段协作的正确姿势

### 3.1 盘前 → 午盘 → 收盘的联动

三个时段不是孤立的，它们形成一个完整的决策闭环：

```
盘前判断 → 午盘验证/修正 → 收盘复盘/总结 → 更新经验 → 下次盘前判断更准
```

**正确的执行流程：**

1. **08:00 盘前**：基于外盘+新闻+盘感，形成「今天大方向」的判断。比如：「外盘偏多，新能源有政策利好，今天看多新能源方向」。

2. **12:00 午盘**：验证盘前判断。如果新能源确实领涨，说明判断正确，给出具体标的；如果新能源没动而AI领涨，说明判断偏差，修正方向。

3. **17:30 收盘**：完整复盘。不管判断对错，都要分析原因。对了：是什么信号让我做出正确判断？错了：是忽略了什么？把这些写进 lessons.md。

### 3.2 常见错误用法

❌ **只用收盘复盘，不用盘前和午盘**。那你就失去了「预判→验证」的环节，复盘变成了「事后诸葛亮」。

❌ **每次都从零分析，不读 lessons.md**。那就永远不会有积累，每天都是新手。

❌ **判断错了不记录**。回避错误不会让你进步。记录错误、分析原因，才是成长的唯一途径。

❌ **只看涨幅榜，不看量能和资金流**。涨幅只告诉你结果，量能和资金流告诉你原因和可持续性。

❌ **量化营业部数据一笔带过**。这是最独特的分析维度，值得花时间研究。

## 四、进阶用法

### 4.1 自定义盘感权重

在 lessons.md 的「盘感权重调整」章节，可以手动记录当前市场更重要的维度：

```markdown
## 盘感权重调整
- 2026年3月: 政策面对市场影响远大于技术面，关注国务院/证监会政策
- 2026年3月: 量化策略近期成功率下降，可能是市场风格切换信号
```

系统在分析时会读取这些权重，自动调整分析侧重。

### 4.2 板块联动追踪

不要只记录一次性的联动关系，要追踪联动链的完整生命周期：

```markdown
- 03/15: 新能源整车启动(比亚迪+8%)
- 03/16: 锂矿跟涨(天齐锂业+5%) ← 联动验证成功
- 03/17: 光伏辅材跟涨(福斯特+4%) ← 联动验证成功
- 03/18: 新能源整车高位震荡，锂矿开始回落 ← 联动链结束信号
```

### 4.3 与你的实际操作结合

最好的用法是：把你的实际操作记录到 lessons.md 中。

```markdown
## 实操记录
| 日期 | 操作 | 标的 | 买入价 | 卖出价 | 盈亏 | 复盘分析 |
|------|------|------|--------|--------|------|----------|
| 03/20 | 买入 | XX科技 | 28.5   | -      | 持仓 | 午盘策略推荐，趋势低吸到MA10 |
```

这样系统就能学习到「什么类型的推荐，实际操作效果如何」。

### 4.4 定期回顾

每周花 10 分钟回顾 lessons.md：
- 这周判断修正了几次？趋势是什么？
- 量化策略有什么新发现？
- 哪些板块联动规律被验证了？
- 盘感权重需要调整吗？

这种定期回顾会让系统的「盘感」快速提升。

## 五、数据完整性保障

### 5.1 多源降级机制

脚本内置了智能降级逻辑：

```
akshare (首选) → 新浪财经 (备用) → 同花顺 (再备用) → 东方财富 (兜底)
```

403/404 错误会立即切换数据源，不会浪费时间重试。指数退避（1.5s→3s→6s→12s→24s）应对网络波动。

### 5.2 如果 akshare 未安装

脚本会自动降级到 HTTP 直连模式，但建议安装 akshare：

```bash
pip install akshare
```

akshare 是免费、开源、覆盖面最广的中国财经数据接口库。它是这套系统的最佳数据源。

### 5.3 数据异常处理

如果某天数据严重缺失（>3个模块失败）：
1. 报告末尾会有「⚠️ 数据采集异常」部分，列出具体失败项
2. 查看 data/lessons.md 的「数据接口问题」章节，是否有历史类似问题
3. 分析时明确标注哪些数据缺失，避免基于不完整数据做判断
4. 如果是 akshare 接口更新导致的，可能需要升级 akshare：`pip install --upgrade akshare`

## 六、钉钉消息的阅读技巧

### 6.1 消息结构

每条钉钉消息都有固定的结构：
- 🔥 今日概况（看这个就够了大概）
- 📈 详情（需要深入时再看）
- 🧠 盘感笔记（最有价值的部分）

### 6.2 阅读优先级

**盘前速递**：重点看「盘前判断」，一句话操作建议。
**午盘策略**：重点看「选股策略」，强势追涨/趋势低吸的具体标的。
**收盘复盘**：重点看「主线板块」+「龙虎榜亮点」+「明日预判」。

### 6.3 符号含义

- 🟢 上涨/流入/看多
- 🔴 下跌/流出/看空
- ⚠️ 风险/异常/需要关注
- 🔥 强势/主线/热门
- 🧠 盘感/经验/判断

## 七、常见问题

**Q: 今天是周末/节假日，系统会怎样？**
A: 数据采集会返回空报告（所有数据为0），但不会报错。非交易日跳过即可。

**Q: 我想关注特定板块怎么办？**
A: 在 lessons.md 的「盘感权重调整」中记录你关注的板块，系统会在分析时重点关注。

**Q: 量化营业部的数据从哪来？**
A: 龙虎榜数据来自东方财富和 akshare。脚本会自动标记机构专用席位和量化特征的席位。

**Q: 如何评估这套系统的准确率？**
A: 看 lessons.md 的「判断修正」章节。如果修正记录在增加，说明系统在学习；如果修正越来越多但原因类似，说明某个维度的分析需要加强。

**Q: 我可以自定义输出格式吗？**
A: 可以。修改 SKILL.md 中的输出格式模板即可。但建议先用默认格式跑一段时间，熟悉后再调整。

## 八、总结

这套系统的核心价值不在于「自动化」，而在于「成长性」。

- 自动化只是基础（每天自动采集数据、自动分析、自动推送）
- 成长性才是核心（通过 lessons.md 的持续积累，分析越来越精准）

用好它的关键就三件事：
1. **三个时段都用**，形成预判→验证→复盘的闭环
2. **每次更新 lessons.md**，让经验有地方安放
3. **定期回顾**，让经验转化为智慧
