# A 股复盘应用钉钉消息适配方案

## 问题背景

您希望修改"应用开发"技能，使其在推送 A 股复盘报告时能够完美适配钉钉的消息格式（图文并茂）。

## 解决方案

由于"应用开发"技能是钉钉内置技能（remote_source: BUILT_IN），无法直接修改其源代码。我们采用以下**三层适配方案**：

### 方案架构

```
┌─────────────────────────────────────────┐
│  第 1 层：应用创建时的 Prompt 注入        │
│  - 在创建 A 股复盘应用时，通过 prompt    │
│    明确指定钉钉消息格式规范             │
│  - 使用 create_a_share_review_app.py   │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  第 2 层：格式化技能模块                 │
│  - a-share-dingtalk-formatter-skill.md │
│  - 提供标准化的消息格式化函数           │
│  - 图片处理策略                         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  第 3 层：测试验证工具                   │
│  - test_dingtalk_message.py            │
│  - 生成示例消息并验证兼容性             │
│  - 生成实际发送命令                     │
└─────────────────────────────────────────┘
```

## 使用方法

### 方法 1：使用创建脚本（推荐）

运行 `create_a_share_review_app.py` 创建应用：

```bash
python create_a_share_review_app.py
```

该脚本会自动在 prompt 中包含钉钉消息格式规范，确保生成的应用具备正确的消息推送能力。

### 方法 2：手动创建应用

在使用"应用开发"技能时，使用以下 prompt：

```
创建一个 A 股复盘分析应用，要求集成钉钉消息推送功能，并严格遵守以下格式规范：

1. 使用 Markdown 格式，但不使用表格（钉钉不支持）
2. 使用列表项展示数据（- **标签**: 值）
3. 标题使用 #/##/### 层级结构
4. 重要数据用**粗体**标注
5. 使用 emoji 提高可读性（📊🎯🏆💰📈📝）
6. 图片单独发送，文字消息中提示"图片已发送"
7. 单条消息不超过 4096 字符

详细模板参考：dingtalk-message-format-guide.md
```

### 方法 3：附加格式化技能

创建应用时引用格式化技能：

```bash
dws aiapp create \
  --prompt "创建 A 股复盘应用，集成钉钉消息格式化能力" \
  --skills a-share-dingtalk-formatter \
  --format json
```

## 钉钉消息格式规范

### ✅ 支持的格式

- 标题：`# H1`, `## H2`, `### H3`
- 粗体：`**文本**`
- 斜体：`*文本*`
- 链接：`[文本](URL)`
- 列表：`-` 或 `1.`
- 引用：`>`
- 代码块：``` ```text ```

### ❌ 不支持的格式

- 表格（Markdown table）
- 内联图片（`![alt](url)`）
- 复杂 HTML

## 图文消息发送策略

### 方案 A：图片 + 文字分开发送（推荐）

```bash
# 步骤 1：发送图片
dws chat message send-by-bot --robot-code "a-share-review-bot" \
  --group "openconv_market_group_123" \
  --title "大盘走势图" \
  --text "https://drive-storage.dingtalk.com/path/to/chart.png" \
  --format json

# 步骤 2：发送文字分析
dws chat message send-by-bot --robot-code "a-share-review-bot" \
  --group "openconv_market_group_123" \
  --title "📊 A 股复盘报告" \
  --text "# 📊 A 股复盘报告\n\n## 大盘概览\n- **上证指数**: 3245.67 (+1.23%)\n..." \
  --format json
```

### 方案 B：使用钉钉文档链接

将完整图文报告生成钉钉文档，然后发送文档链接：

```markdown
👉 **查看详细图文报告**: https://doc.dingtalk.com/doc/xxx
```

## 注意事项

1. **消息长度**：单条 Markdown 消息不超过 4096 字符
2. **发送频率**：机器人发消息频率不超过 20 条/分钟
3. **图片 URL**：优先使用钉盘永久链接
4. **特殊字符**：双引号、换行符需要正确转义
5. **@功能**：添加 `--at-users userId` 参数可@特定用户

## 常见问题

**Q: 为什么不直接修改"应用开发"技能？**

A: "应用开发"是钉钉内置技能（BUILT_IN），源代码不在本地，无法直接修改。通过在 prompt 中注入规范和创建辅助技能模块，可以达到同样的效果。

**Q: 如何让应用自动发送消息到钉钉？**

A: 在创建应用时明确要求集成钉钉机器人 API，并提供 robot-code 和 group-id 配置。应用会生成相应的发送命令。

**Q: 图片无法显示怎么办？**

A: 确保图片 URL 可公开访问。推荐使用钉盘存储图片，或使用临时 CDN URL（注意有效期）。

**Q: 消息被截断怎么办？**

A: 检查消息长度是否超过 4096 字符。如超限，简化内容或拆分为多条消息发送。
